# Fix-2 Design Spec
**Date:** 2026-04-09  
**Scope:** 12 improvements to Folia — app shell, editor UX, markdown editor

---

## Overview

Fix-2 covers three areas:
1. **App Shell** — settings modal, theme toggle, empty state, note active state, Ctrl+K command palette
2. **Editor** — mode switch redesign, tag editor, download dropdown
3. **Markdown Editor** — rendered preview, dark mode colors, formatting toolbar with colors

All changes target the existing SvelteKit + Dexie.js codebase. No new routes are added; one route (`/settings`) is removed.

---

## Section 1: App Shell

### 1.1 Settings Modal (replaces `/settings` route)

- Delete `src/routes/settings/+page.svelte` and the `settings/` directory.
- Create `src/lib/components/SettingsModal.svelte` with all current settings content: theme, language, editor default, AI key management, backup/restore (JSON export/import).
- Mount `SettingsModal` in `+layout.svelte` behind a `showSettings` boolean.
- Add `showSettings` writable boolean to `src/lib/stores/ui.js`.
- The sidebar footer settings button sets `showSettings.set(true)`; layout reads `$showSettings`.
- The modal renders a full-screen backdrop (`rgba(0,0,0,0.4)`), centered panel (max-width 560px), closeable via Escape key or backdrop click.
- All existing logic (`setTheme`, `setLang`, `setEditorDefault`, `exportAll`, `importJson`, API key management) moves into the modal component.

### 1.2 Theme Toggle in Sidebar Footer

- Add a sun/moon icon button in `.sidebar-footer` (left of the settings button).
- Clicking it toggles between `'light'` and `'dark'` via the existing `theme` store and `db.prefs`.
- Uses `Sun` / `Moon` icons from `lucide-svelte`.
- Same visual style as existing `.footer-btn`.

### 1.3 Empty State (homepage)

- `src/routes/+page.svelte`: replace the static "Nessuna nota" paragraph with a randomly selected message from a pool.
- Message pool defined in `en.json` and `it.json` under key `emptyState.messages` (array of strings, 3 categories: welcome, tips, quotes — ~9 messages total per language).
- On component mount, pick `messages[Math.floor(Math.random() * messages.length)]`.
- Display centered, `font-style: italic`, `color: var(--color-text-muted)`, max-width 360px.

### 1.4 NoteCard Active State

- In `NoteCard.svelte`, when `active=true`:
  - Add `border-left: 2px solid var(--color-accent)` on the card container.
  - Slightly more prominent background: `background: var(--color-surface)` (vs transparent when inactive).
- Ensure the left border doesn't shift layout (use `border-left` + compensating `padding-left` reduction).

### 1.5 Ctrl+K Command Palette

- Create `src/lib/components/CommandPalette.svelte`.
- Mount in `+layout.svelte` behind a `showPalette` boolean.
- `+layout.svelte` `handleKeydown` handler: if `(e.ctrlKey || e.metaKey) && e.key === 'k'`, prevent default and set `showPalette = true`.
- UI: full-screen backdrop, centered panel (max-width 560px), search input auto-focused on open.
- As the user types, run `searchNotes(query)` from `search.js` and show up to 8 results.
- Each result row shows: note title (bold), excerpt (truncated to ~60 chars), folder name (if any), tags as small badges.
- Click on a result: `goto('/note/${result.id}')` + close palette.
- Escape closes. Keyboard navigation (↑↓ arrows + Enter) to select result.
- Empty query shows the 8 most recently updated notes as default list.

## Section 2: Editor

### 2.1 Mode Switch Redesign

- In `EditorToolbar.svelte`, replace the plain "MD"/"Rich" text button with a pill toggle.
- Two segments side by side: paintbrush icon (rich) | `</>` icon (markdown), from `lucide-svelte` (`Paintbrush`, `Code2`).
- Active segment has a sliding background using CSS transition (`transform` on a pseudo-element or absolute positioned indicator).
- Remains positioned at the right side of the toolbar (`margin-left: auto`).

### 2.2 Tag Editor Above Title

- In `Editor.svelte`, add a tag row above the `<input class="title-input">`.
- Shows existing `note.tags` as badge chips, each with a `×` button to remove.
- A `+` button opens an inline input with a dropdown showing autocomplete suggestions from `allTags` — a local `liveQuery(() => db.notes.filter(n => !n.deletedAt).toArray())` in `Editor.svelte` that extracts the unique tag set.
- Autocomplete filters existing tags by prefix match.
- Selecting a suggestion or pressing Enter adds the tag; saves immediately via `db.notes.update(note.id, { tags: [...note.tags, newTag], updatedAt: Date.now() })`.
- Removing a tag saves immediately.
- Tags are stored as the existing `tags: string[]` field on notes.

### 2.3 Download Dropdown

- In `EditorToolbar.svelte`, add a download button (lucide `Download` icon) to the right of the AI button.
- Click toggles a small dropdown with two items: "Scarica .md" and "Scarica .txt".
- The toolbar dispatches a `downloadNote` event with `{ detail: { format } }` ('md' or 'txt').
- `Editor.svelte` handles the event, using `noteToMarkdown(note)` / `noteToText(note)` from `export.js` with the local `note` object.
- Dropdown closes on selection or click outside.

## Section 3: Markdown Editor

### 3.1 Rendered Preview

- Install `marked` (`npm install marked`).
- In `Editor.svelte`, replace the `<pre class="md-preview-text">` with a `<div class="md-preview-rendered">`.
- Use `{@html marked.parse(markdownContent || '')}` to render content.
- Reactive: `$: renderedHtml = marked.parse(markdownContent || '')` updates on every input.
- Configure `marked` with `{ breaks: true, gfm: true }`.
- Add `:global(.md-preview-rendered)` CSS rules for `h1–h6`, `strong`, `em`, `ul`, `ol`, `li`, `code`, `pre`, `blockquote`, `a` — all using `--color-text`, `--color-text-muted`, `--color-accent` variables.
- Configure marked with `marked.use({ gfm: true, breaks: true })` and use `marked.parse()` with default options. Since `marked` renders HTML as-is by default, color spans (`<span style="color:…">`) will render correctly without extra configuration.

### 3.2 Dark Mode Color Fixes

- `.md-source` textarea: explicit `color: var(--color-text); background: var(--color-surface);` (not `transparent`/`inherit`).
- `.md-preview-rendered`: `color: var(--color-text); background: transparent;`.
- All `:global()` preview element styles use CSS variables, no hardcoded colors.
- Verify title input and tag row also use `color: var(--color-text)` explicitly.

### 3.3 Markdown Toolbar with Colors

In `EditorToolbar.svelte`, when `mode === 'markdown'`:

**Formatting buttons** (insert syntax at cursor / wrap selection):
- **B** → wraps selection in `**...**`
- _I_ → wraps selection in `*...*`
- H1, H2, H3 → prepends `# `, `## `, `### ` to current line
- `• Lista` → prepends `- ` to current line
- `</>` → wraps selection in `` `...` ``

These require the toolbar to receive a reference to the markdown `<textarea>` (passed as a prop or via event) to manipulate selection and value.

**Color picker section:**
- 8 fixed palette colors (e.g., red, orange, yellow, green, teal, blue, purple, pink — accessible contrast in both themes).
- 1 native `<input type="color">` picker button.
- Last 4 custom colors saved in `db.prefs` under key `customColors` (array of hex strings, max 4, FIFO).
- Selecting any color wraps the current textarea selection in `<span style="color:#xxxxxx">...</span>`.
- Custom color from picker is saved to `customColors` pref and shown in the 4 saved slots.
- Total UI elements in color section: 8 fixed + 4 saved + 1 picker button = 13 (≈12 as requested).

The textarea ref is passed as a prop `mdTextarea` to `EditorToolbar`, bound in `Editor.svelte` via `bind:this`.

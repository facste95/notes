# Folia — Fix-1 Design Spec

**Date:** 2026-04-08  
**Scope:** 10 bug fixes and features from `fix-1.md`  
**Approach:** Two-phase delivery — bug fixes first, then features.

---

## Phase 1 — Bug Fixes

### 1. Sidebar open/close (item 1)

**Problem:** The sidebar toggle button or its CSS transition is not working correctly.

**Fix:** Verify that the toggle button in `Sidebar.svelte` calls `sidebarOpen.update(v => !v)` correctly. Audit the `.closed` CSS class transition (width: 260px → 40px, 0.3s ease) for conflicts with other styles. No architectural changes.

---

### 2. Save before mode toggle (item 3)

**Problem:** Switching between rich and markdown modes while the 800ms debounce is pending causes the latest changes to be lost.

**Fix:** In `Editor.svelte`, when the `toggleMode` event fires from `EditorToolbar`, flush the pending save synchronously (cancel debounce timer + immediate `db.notes.update()`) before applying the mode switch.

---

### 3. Theme font colors (item 6)

**Problem:** Some components use hardcoded colors (e.g., `color: black`, `color: #333`) instead of CSS custom properties, causing incorrect appearance in dark mode.

**Fix:** Audit all components and TipTap editor CSS. Replace hardcoded color values with CSS custom properties (`var(--color-text)`, `var(--color-text-muted)`, `var(--color-bg)`, etc.).

---

### 4. Note click/edit broken (item 7)

**Problem:** Clicking a note card does not open it for editing.

**Fix:** Investigate the click handler in `NoteCard.svelte`, the `note/[id]/+page.svelte` route load, and `db.notes.get(id)`. Identify and fix the root cause preventing navigation or note loading.

---

## Phase 2 — Features

### 5. Settings modal in toolbar (item 2)

**Location:** `EditorToolbar.svelte` — new `Settings` icon button on the right side of the toolbar.

**Behavior:** Opens `SettingsModal.svelte` — an inline modal overlay. Closes on ESC or click-outside.

**Modal contents:**
- Theme toggle: Light / Dark (saves to `db.prefs` key `'theme'`, updates `theme` store)
- Language toggle: Italian / English (saves to `db.prefs` key `'language'`, updates `language` store)
- Default editor mode: Rich / Markdown (saves to `db.prefs` key `'editorMode'`)
- Link: "Impostazioni avanzate" → navigates to `/settings`

**New file:** `src/lib/components/SettingsModal.svelte`

---

### 6. Toolbar active states (item 4)

**Problem:** Toolbar formatting buttons (Bold, Italic, H1, H2, H3, Bullet list) show no visual feedback when the cursor is inside formatted text.

**Fix:** In `EditorToolbar.svelte`, use `editor.isActive('bold')`, `editor.isActive('italic')`, `editor.isActive('heading', { level: N })`, `editor.isActive('bulletList')` to conditionally apply an `.active` CSS class to each button. Subscribe to TipTap's `selectionUpdate` and `transaction` events to trigger reactivity.

---

### 7. Folder drag-and-drop (item 5)

**Library:** `svelte-dnd-action`

**Behavior:**
- Note cards in the sidebar become draggable items.
- Folder items and the "Tutte le note" section become drop zones.
- On drop into a folder: `db.notes.update(noteId, { folderId: targetFolderId })`.
- On drop into "Tutte le note": `db.notes.update(noteId, { folderId: null })`.
- Visual feedback: highlight the target folder/section while dragging over it.
- Notes can also be dragged to the Trash section (see below).

---

### 8. Trash bin — soft delete (items 8, 9, 10)

#### DB Schema Migration

Dexie `version(2)` migration adds:
- `notes` table: new `deletedAt` field (timestamp | null), new index on `deletedAt`
- `folders` table: new `deletedAt` field (timestamp | null), new index on `deletedAt`

All existing queries for notes and folders add `.filter(item => !item.deletedAt)`.

#### Auto-cleanup on app mount

In `+layout.svelte` on mount: query notes and folders where `deletedAt < (Date.now() - 30 * 24 * 60 * 60 * 1000)` and permanently delete them via `db.notes.bulkDelete()` / `db.folders.bulkDelete()`.

#### Sidebar Trash section

A new "Cestino" section appears at the bottom of `Sidebar.svelte`, visible only when at least one deleted item exists.

**Contents:**
- "Svuota cestino" button — permanently deletes all items with `deletedAt != null`.
- List of deleted notes and folders.
- Per-item buttons: "Ripristina" (sets `deletedAt = null`) and "Elimina definitivamente" (hard delete).

#### Deleting notes and folders

- Deleting a note: sets `note.deletedAt = Date.now()` (soft delete) instead of removing from DB.
- Deleting a folder: sets `folder.deletedAt = Date.now()`. All notes inside the folder are also soft-deleted (set their `deletedAt` too).
- Drag-to-trash: dragging a note or folder onto the Trash section in the sidebar triggers soft delete.

#### Restoring

- Restoring a note: sets `note.deletedAt = null`.
- Restoring a folder: sets `folder.deletedAt = null` and also restores all notes that were deleted at the same time as the folder (within a 1-second window of the folder's `deletedAt` timestamp).
